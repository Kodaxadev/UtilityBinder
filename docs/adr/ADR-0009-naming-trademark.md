# ADR-0009: Naming — "UtilityBinder" adopted; formal knockout search still required before print

**Status:** Accepted (supersedes the "SiteSentry Records" working title, 2026-06-11)
**Date:** 2026-06-11
**Update 2026-06-11:** Knockout executed — USPTO Trademark Search wordmark query for "UtilityBinder" returned **0 results**. Name cleared for pilot use, printed materials, and LLC/DBA. Revisit mark strength only if/when marketing scales (the name can be changed later if needed; artifacts are dated and versioned).
**Evidence:** [market-validation.md](../research/market-validation.md) §11

## Context

"SiteSentry" came from the original brainstorm with no clearance work, and the "site sentry / site monitoring" naming space is crowded (construction-site cameras, web monitoring). A naming review evaluated candidates against three criteria: (1) says what it is to a 60-year-old park owner, (2) survives being said over the phone, (3) no locate/dig/survey/safety implication ([ADR-0002](ADR-0002-liability-posture-not-a-locate.md)/[0003](ADR-0003-patent-risk-avoidance.md)).

### Candidates evaluated

| Name | Verdict | Reason |
|------|---------|--------|
| **UtilityBinder** | **Adopted** | Covers all utility types (not valve/water-coded); sounds operational, not techy; names the replacement behavior — the office binder; light screen shows no exact-match software competitor (mostly physical tackle/fishing binder products); expands cleanly to marinas, fairgrounds, retreats |
| Shutoff Atlas | Fallback | Clear, but too valve/water-coded and too "map product" — the product is records, not an atlas |
| Shutoff Key | Maybe | Good emergency phrase but sounds like a physical valve tool |
| ControlMap | Reject | Existing GRC platform (ScalePad); too close to GIS/control language |
| ControlPoint | Reject | Crowded mark; "control point" is Esri's term for an accurately surveyed coordinate — exactly the lane to avoid |
| ControlVault | Reject | Dell ControlVault exists; reads cybersecurity/hardware |
| CampKey | Reject | CampKey/Campivio already exists in campground management |
| FieldBinder | Usable but weaker | Generic, less tied to utilities |
| ParkBinder | Too narrow | Bad for marinas/fairgrounds/retreat centers |
| GroundBinder | Reject | Existing construction-material mark; excavation-adjacent sound |

## Decision

1. **Product/company name: UtilityBinder.** All documents, copy, and artifacts use it effective immediately.
2. The name is deliberately broad and operational, so it does **not** carry the emergency urgency itself — **the headline must**: *"Know what to shut off when Dave isn't there."* Name and hook are a paired decision; neither is changed without the other being re-evaluated.
3. **Brand architecture (canonical artifact names):**
   - Pilot offer: **Emergency Utility Map Pilot**
   - Primary artifact: **UtilityBinder Emergency PDF**
   - Internal worksheet: **Dave Sheet** (knowledge-capture worksheet)
   - Hosted map: **Staff Emergency Map**
   - Contractor export: **Emergency Reference Packet**
   - Core sales sentence: *"We build your UtilityBinder from your old maps, staff memory, photos, and repair notes."*
4. **Formal knockout before anything irreversible** (LLC name, trademark filing, printed materials, paid domain portfolio): a light web screen is NOT legal clearance. Use the **USPTO Trademark Search system** (TESS was retired in 2023 — do not reference TESS in checklists). Per USPTO guidance, exact-wording lookup is only the first "knock-out" step: search `CM:"utility binder"` and expanded variants (combined marks, plurals, phonetic equivalents, "binder" + utility-adjacent terms) in classes 9 & 42, plus state business registry and domain/social checks. Budget one day.
5. The LLC ([ADR-0002](ADR-0002-liability-posture-not-a-locate.md) §5) may be formed under a neutral name with UtilityBinder as a DBA, decoupling legal formation from the mark.

## Consequences

- Discovery outreach proceeds immediately under UtilityBinder (conversation-stage use is low risk).
- "UtilityBinder" risk profile: the term has generic physical-product usage (tackle binders), which weakens trademark strength but also lowers collision risk in software classes — acceptable for this stage; revisit mark strength only if the business scales enough to care.
- All docs renamed in place; "SiteSentry" survives only in this ADR's history and the evidence log's naming entry.
